# LeetCode Go 学習リポジトリ

## 学習者について

- **プログラミング経験**: ゼロからのスタート（初めてのプログラミング）
- **使用言語**: Go
- **学習方法**: LeetCode問題 + アルゴリズム書籍

## このリポジトリの目的

1. LeetCode の問題を Go で解く
2. アルゴリズムとデータ構造の基礎を身につける
3. Go 言語そのものの書き方に慣れる

## ディレクトリ構成

```
leetcode/
├── CLAUDE.md
├── README.md
├── go.mod                    # Go モジュール設定
├── easy/                     # LeetCode 難易度別
│   └── 0001_two_sum/
│       ├── solution.go
│       └── solution_test.go
├── medium/
├── hard/
└── notes/                    # アルゴリズムのメモ
```

## Claude へのお願い

### 説明スタイル
- プログラミング初心者向けに説明する
- 専門用語は使ったあと必ず日本語で補足する
- コードを書く前に「何をしているか」を日本語で説明する
- なぜその書き方をするのか、理由まで教える

### 学習サポート
- 答えをすぐ教えるより、ヒントを与えて自分で考えさせることを優先する
- 解けた後は「別の解き方」や「計算量（Big O）」の話も加える
- Go 固有の書き方（slice, map, goroutine など）が出てきたら都度解説する

### コマンド実行のルール（重要）
- **Claude はコマンドを自分で実行しない**
- コマンドが必要なときは、以下の形式でユーザーに伝えてから実行してもらう：

  ```
  【目的】何のためにこのコマンドを使うか
  【コマンド】実行するコマンド
  【解説】コマンドの各部分が何をしているか
  ```

- コマンドの実行結果もユーザーの知識になるため、結果の読み方も合わせて説明する
- 環境構築・テスト実行・ビルドなど、あらゆるコマンドに対してこのルールを適用する

### コードスタイル
- `gofmt` に従ったフォーマットで書く
- テストコードも必ずセットで作成する（`go test` で動作確認できる状態）
- コメントは日本語で書いてよい

## 問題ファイルの命名規則

```
{難易度}/{4桁番号}_{snake_case_問題名}/solution.go
例: easy/0001_two_sum/solution.go
```

## テスト実行

```bash
# 特定の問題のテスト
go test ./easy/0001_two_sum/

# 全テスト
go test ./...
```
